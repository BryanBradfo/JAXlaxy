# JAXlaxy Observatory

Interactive 3D map of the JAX ecosystem — a cinematic companion to the main [`../README.md`](../README.md).
Each entry becomes a glowing star. Size = GitHub stars. Color = health status (🟢 active · 🟡 stable · 🔴 legacy).

## Dev

```bash
# First-time: generate the galaxy data (from repo root)
GITHUB_TOKEN="$(gh auth token)" python ../scripts/health_check.py --json public/galaxy.json

# Then from inside observatory/
npm install
npm run dev        # → http://localhost:4321
```

`npm run build` emits static files to `dist/`. No server runtime required.

## Stack

- **Astro 6** + React islands (only the 3D canvas ships JS; the rest is static HTML/CSS)
- **Tailwind CSS v4** via the `@tailwindcss/vite` plugin (CSS-first `@theme` tokens — see `src/styles/global.css`)
- **Three.js** via `@react-three/fiber`, `@react-three/drei`, `@react-three/postprocessing` (bloom glow)

## Architecture

```
src/
├── pages/          → index.astro (galaxy), table.astro (stub)
├── layouts/        → Base.astro (HTML shell, fonts, meta)
├── components/
│   ├── Galaxy.tsx    → <Canvas> + camera + lights + bloom + OrbitControls
│   ├── StarField.tsx → positions stars deterministically by section + hash
│   └── Star.tsx      → individual mesh; hover + click + color-by-status
├── types/galaxy.ts → TS shape matching scripts/health_check.py --json output
└── styles/global.css → Tailwind v4 import + @theme palette
```

## Data pipeline

The site loads `/galaxy.json` (from `public/galaxy.json`) at runtime. That file is
generated by `scripts/health_check.py` — which is the same script that runs in
the weekly Autopilot workflow. Running `--json` alongside a regular health audit
produces both the markdown report and the Observatory's data in one pass.

## Two spots for user taste

These are flagged with `TODO(user-contribution)` in the source — tweak freely:

1. **`hashToPosition(...)`** in `src/components/StarField.tsx` — defines the
   galaxy's overall shape. Latitude bands per section today; could be spiral arms,
   orbital rings, or anything else.
2. **`STATUS_PALETTE`** in `src/components/Star.tsx` — exact hex + bloom intensity
   per status. Pure primaries blow out under bloom; the defaults are "bloom-safe"
   starting points.
